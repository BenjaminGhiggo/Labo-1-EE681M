Sorter por metodo de burbuja (Bubble Sort) — Modelo FSM+D en VHDL

Dispositivo: Cyclone V SoC 5CSEMA5F31C6 | Reloj: 12 ns | Quartus Prime Lite 18.1

1. Objetivo

Disenar, simular e implementar un circuito secuencial sincrono que ordene k=8 numeros sin signo de n=4 bits almacenados en registros R0 a R7, utilizando el metodo de la burbuja (Bubble Sort), en forma ascendente (menor en R0, mayor en R7).

El diseno sigue el modelo FSM+D (Maquina de Estados Finitos + Datapath), descrito jerarquicamente en VHDL con tres archivos: sorter.vhd (tope), datapath.vhd y controlador.vhd.

2. Diagrama de Bloques del Sistema

Figura 1 — Diagrama de bloques del sorter: interconexion entre datapath y controlador.
Modelo FSM+D: El controlador (FSM) genera senales de control que dirigen las operaciones del datapath. Las senales internas i, j y control son las unicas conexiones entre ambos bloques. Este modelo separa la logica de control de la logica de datos, facilitando el diseno y verificacion.

3. Especificacion del Datapath

3.1 Descripcion funcional

El datapath contiene un banco de 8 registros (R0 a R7), cada uno de 4 bits, implementados con flip-flops tipo D activados por flanco de subida. Realiza tres operaciones segun control[2..0]:

control[2..0]OperacionDescripcion
"001"Borrado sincronoTodos los registros R0..R7 se ponen a "0000"
"010"EscrituraSi wrinit='1', escribe datain en R[radd]
"100"Comparar/IntercambiarSi R[i] > R[j], intercambia sus contenidos (swap)
"000"Sin operacionLos registros mantienen su valor actual

3.2 Diagrama de hardware del Datapath

Figura 2 — Hardware del datapath: banco de registros (flip-flops D), MUX de lectura, comparador y logica de swap.
Flip-Flops tipo D: Cada bit de cada registro se implementa con un flip-flop D activado por flanco de subida. 8 registros x 4 bits = 32 flip-flops D solo para el banco. La salida dataout es combinacional (MUX 8:1 selecciona R[radd] sin pasar por registro).

3.3 Codigo VHDL del Datapath (comentado)

library ieee;
use ieee.std_logic_1164.all;
use ieee.numeric_std.all;

-- Entidad: define puertos de entrada/salida del bloque
entity datapath is
  port (
    clock   : in  std_logic;                     -- Reloj, activo en flanco de subida
    wrinit  : in  std_logic;                     -- Habilitacion de escritura
    control : in  std_logic_vector(2 downto 0); -- Operacion: 001=borrar, 010=escribir, 100=swap
    i, j    : in  std_logic_vector(2 downto 0); -- Indices para burbuja (del controlador)
    datain  : in  std_logic_vector(3 downto 0); -- Dato a escribir (4 bits, rango 0-15)
    radd    : in  std_logic_vector(2 downto 0); -- Direccion del registro (0 a 7)
    dataout : out std_logic_vector(3 downto 0)  -- Lectura combinacional de R[radd]
  );
end datapath;

architecture behav of datapath is
  -- Tipo arreglo: 8 registros de 4 bits (cada bit = 1 flip-flop D)
  type file_register is array(0 to 7) of std_logic_vector(3 downto 0);
  signal R: file_register;

  -- Senales auxiliares: convierten std_logic_vector a entero
  -- (VHDL requiere enteros para indexar arreglos)
  signal i_int, j_int, r_int : integer range 0 to 7;
begin
  -- Conversiones combinacionales (sin flip-flop, instantaneas)
  i_int <= to_integer(unsigned(i));
  j_int <= to_integer(unsigned(j));
  r_int <= to_integer(unsigned(radd));

  -- PROCESO SINCRONO: todo ocurre en el flanco de subida del reloj
  -- (sin reset asincrono, segun indicacion del profesor)
  process (clock)
  begin
    if (clock'event and clock = '1') then

      -- BORRADO SINCRONO: limpia los 8 registros
      if control = "001" then
        R(0) <= "0000"; R(1) <= "0000";
        R(2) <= "0000"; R(3) <= "0000";
        R(4) <= "0000"; R(5) <= "0000";
        R(6) <= "0000"; R(7) <= "0000";

      -- ESCRITURA: solo si wrinit esta activo
      elsif control = "010" then
        if wrinit = '1' then
          R(r_int) <= datain;  -- Guarda datain en el registro apuntado por radd
        end if;

      -- COMPARAR E INTERCAMBIAR: corazon del bubble sort
      -- Compara como numeros sin signo (unsigned)
      elsif control = "100" then
        if unsigned(R(i_int)) > unsigned(R(j_int)) then
          R(i_int) <= R(j_int);  -- R[i] recibe el valor menor
          R(j_int) <= R(i_int);  -- R[j] recibe el valor mayor
          -- Nota: en VHDL, ambas asignaciones usan los valores
          -- ANTES del flanco, por lo que el swap es atomico
        end if;
      end if;
    end if;
  end process;

  -- SALIDA COMBINACIONAL (no registrada): MUX 8:1
  -- Muestra el contenido de R[radd] instantaneamente
  dataout <= R(r_int);
end behav;

4. Especificacion del Controlador (FSM)

4.1 Descripcion funcional

El controlador es una Maquina de Estados Finitos (FSM) tipo Moore: las salidas dependen solo del estado actual (no de las entradas directamente). Esto evita glitches y problemas de timing.

Implementado con el modelo de dos procesos:

  • Proceso 1 (secuencial): Flip-flops D para el registro de estado, i_reg y j_reg. Reset asincrono.
  • Proceso 2 (combinacional): Logica de siguiente estado y codificacion de salidas. Sin memoria.

4.4 Codigo VHDL del Controlador (comentado)

library ieee;
use ieee.std_logic_1164.all;
use ieee.numeric_std.all;

entity controlador is
  port (
    clock   : in  std_logic;                     -- Reloj del sistema
    resetn  : in  std_logic;                     -- Reset asincrono activo bajo
    go      : in  std_logic;                     -- '1'=inicializacion, '0'=ordenar
    i, j    : out std_logic_vector(2 downto 0); -- Indices al datapath
    control : out std_logic_vector(2 downto 0); -- Operacion al datapath
    done    : out std_logic                      -- '1' al terminar
  );
end controlador;

architecture behav of controlador is
  -- 10 estados para el algoritmo bubble sort
  type state_type is (S0, S1, S2, S3, S4, S5, S6, S7, S8, S9);
  signal estado_actual, estado_sig : state_type;
  -- Contadores de indices (3 flip-flops D cada uno)
  signal i_reg, j_reg   : unsigned(2 downto 0);
  signal i_next, j_next : unsigned(2 downto 0);
begin
  -- PROCESO 1: Registro de estado (secuencial)
  -- Implementa los flip-flops D con reset asincrono
  process (clock, resetn)
  begin
    if resetn = '0' then            -- Reset asincrono: vuelve a S0
      estado_actual <= S0;
      i_reg <= (others => '0');
      j_reg <= (others => '0');
    elsif clock'event and clock = '1' then  -- Flanco de subida
      estado_actual <= estado_sig;   -- Carga siguiente estado
      i_reg <= i_next;              -- Carga siguiente i
      j_reg <= j_next;              -- Carga siguiente j
    end if;
  end process;

  -- PROCESO 2: Logica combinacional (sin flip-flops)
  -- Calcula el siguiente estado y codifica las salidas Moore
  process (estado_actual, go, i_reg, j_reg)
  begin
    -- Valores por defecto (previene latches en sintesis)
    estado_sig <= estado_actual;
    i_next <= i_reg;  j_next <= j_reg;
    control <= "000";  done <= '0';
    i <= std_logic_vector(i_reg);
    j <= std_logic_vector(j_reg);

    case estado_actual is
      when S0 => -- Esperar go='1' para empezar
        if go = '1' then estado_sig <= S1; end if;

      when S1 => -- Borrar todos los registros
        control <= "001";
        i_next <= (others => '0');
        j_next <= (others => '0');
        estado_sig <= S2;

      when S2 => -- Modo escritura, esperar go='0'
        control <= "010";
        if go = '0' then
          i_next <= (others => '0');
          j_next <= to_unsigned(1, 3);
          estado_sig <= S3;
        end if;

      when S3 => -- Preparar j = i + 1
        j_next <= i_reg + 1;
        estado_sig <= S4;

      when S4 => -- Enviar orden de comparar/swap al datapath
        control <= "100";
        estado_sig <= S5;

      when S5 => -- Verificar limite de j (DESPUES de comparar)
        if j_reg = 7 then estado_sig <= S7;
        else estado_sig <= S6; end if;

      when S6 => -- Incrementar j y volver a comparar
        j_next <= j_reg + 1;
        estado_sig <= S4;

      when S7 => -- Incrementar i
        i_next <= i_reg + 1;
        estado_sig <= S8;

      when S8 => -- Si i=7 (todos los pares procesados), terminar
        if i_reg = 7 then estado_sig <= S9;
        else estado_sig <= S3;
        end if;

      when S9 => -- Ordenamiento completo
        done <= '1';
        estado_sig <= S9; -- Queda aqui permanentemente

      when others => estado_sig <= S0;
    end case;
  end process;
end behav;

6. Archivo Tope: sorter.vhd

Conecta los dos bloques mediante senales internas. Es puramente estructural (sin logica propia):

-- Senales internas (cables entre controlador y datapath)
signal ctrl_signal : std_logic_vector(2 downto 0); -- control[2..0]
signal i_signal    : std_logic_vector(2 downto 0); -- indice i
signal j_signal    : std_logic_vector(2 downto 0); -- indice j

-- Instancia del datapath: recibe control, i, j del controlador
datos: datapath port map (
  clock => clock, wrinit => wrinit,
  control => ctrl_signal,    -- del controlador
  i => i_signal, j => j_signal,  -- del controlador
  datain => datain, radd => radd, dataout => dataout
);

-- Instancia del controlador: genera control, i, j, done
control: controlador port map (
  clock => clock, resetn => resetn, go => go,
  i => i_signal, j => j_signal,  -- hacia el datapath
  control => ctrl_signal,    -- hacia el datapath
  done => done
);
